# 🎯 소개팅각 MVP 기획안

## 🎯 미션 (Mission)

**"내가 원하는 소개팅을, 늦지 않게 발견하게 한다"**

오프라인 소개팅은 여전히 활발하지만
정보는 흩어져 있고, 조건은 제각각이며,
알았을 땐 이미 마감된 경우가 많습니다.

소개팅각은 '의지'가 아니라 '탐색 시스템'으로
사람들이 자기 타입의 소개팅을 제때 발견하도록 돕습니다.

---

## 📝 MVP 제품 정의

소개팅각은
여러 업체에 흩어진 오프라인 소개팅 이벤트 정보를 한 곳에 모아,
유저가 조건·지역·지도 기반으로 빠르게 탐색할 수 있도록 돕는 플랫폼입니다.

### 핵심 가치

- **"많이 보여주기"가 아니라 "맞는 것만 보여주기"**
- **정보 나열이 아니라 발견 경험**

---

## 🔍 해결할 문제 (Problem)

### 유저 관점

- 오프라인 소개팅 이벤트 정보가 여기저기 흩어져 있어 찾기 어렵다
- 내가 원하는 타입/조건의 소개팅을 찾기 힘들다
  (컨셉, 나이대, 지역, 음주 여부 등)
- 집·회사 근처에서 열리는 소개팅을 알기 어렵다
- 정보를 발견해도 이미 예약이 마감된 경우가 많다

### 업체 관점

- 소개팅 이벤트를 열어도 참여할 유저 풀 확보가 어렵다
- 홍보 채널이 분산되어 노출 효율이 낮다
- 이벤트의 조건(지역, 날짜, 컨셉 등)을 명확히 설명할 공간이 부족하다

### 핵심 문제 요약

> **"소개팅을 찾는 사람과 소개팅을 여는 업체를
> 효율적으로 연결해주는 탐색 허브가 없다"**

---

## 👥 타겟 사용자 (MVP 기준)

### 1차 타겟

- 20–30대 직장인
- 오프라인 소개팅에 거부감은 없지만 직접 발품 팔아 찾기엔 귀찮은 사람
- "아무 소개팅"이 아니라 조건이 맞는 소개팅을 원하는 사람

### 2차 타겟

- 정기적으로 소개팅 이벤트를 여는 소개팅 업체 / 커뮤니티 운영자

---

## 💡 MVP 핵심 기능

### 1. 소개팅 탐색 (핵심)

#### 지도 + 리스트 탐색

**지도 뷰**

- 내 주변에서 열리는 소개팅 이벤트를 한눈에 확인

**리스트 뷰**

- 카드 형태로 조건·가격·날짜를 빠르게 비교

### 2. 필터 (MVP의 중심 기능)

#### 조건 기반 탐색

| 필터 항목        | 설명                        |
| ---------------- | --------------------------- |
| 날짜             | 이벤트 날짜                 |
| 지역             | 시 / 구                     |
| 로테이션 여부    |                             |
| 지자체 연계 여부 |                             |
| 음주 여부        | 커피 / 와인                 |
| 시간             |                             |
| 인원수           |                             |
| 나이대           |                             |
| 돌싱 여부        |                             |
| 종교             |                             |
| 컨셉             | 솔로지옥, 환승연애, 남매 등 |
| 애프터파티 여부  |                             |
| 요일             |                             |
| 온라인 여부      |                             |
| 키 제한 여부     |                             |
| 단발성 여부      |                             |
| 가격대           |                             |

**목표:**

> "이 소개팅, 나한테 맞나?"를 필터에서 이미 결정하게 만든다

### 3. 큐레이션

- **인기순**: 클릭 수 기준
- **지역 기반 큐레이션**
  - 해당 지역에 이벤트가 없으면 노출하지 않음
- **전체 보기 / 조건별 빠른 선택**

### 4. 소개팅 카드 UI

카드에 노출되는 정보:

| 항목          | 설명                                                  |
| ------------- | ----------------------------------------------------- |
| 썸네일 이미지 |                                                       |
| 제목          |                                                       |
| 가격          | 여러 가격일 경우 가장 저렴한 가격 표시                |
| 단발성 여부   |                                                       |
| 날짜/요일     | 단발성(Y): 가장 가까운 날짜 표시 / 정기(N): 요일 표시 |

### 5. 상세 정보 & CTA

- 제목
- 인기 여부
- 가격
- 썸네일
- 업체명
- 지역 (시 / 구)
- 태그 정보
  (커피/와인, 지자체, 로테이션, 돌싱, 종교, 컨셉, 애프터파티, 온라인, 키 제한 등)
- **CTA: 방문하기**
  - 업체 신청 페이지로 이동

## 🧭 유저 스토리 (MVP)

> 유저 A는 오늘도 짝을 찾기 위해 소개팅 정보를 검색하다가
> 우연히 '소개팅각'을 알게 되어 방문한다.
>
> 지도와 필터를 통해
> "이번 주말 · 집 근처 · 커피 소개팅" 조건을 선택하고,
> 맞는 이벤트를 빠르게 발견한다.
>
> 상세 정보를 확인한 뒤
> 별도의 탐색 없이 바로 신청 페이지로 이동한다.
